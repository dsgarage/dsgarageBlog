# See-through: アニメキャラクターの単一画像レイヤー分解

## 概要

本論文は、**静止したアニメイラスト1枚**から、操作可能な **2.5D モデル**へ自動変換するフレームワーク「See-through」を提案する。従来のプロフェッショナルワークフローでは、手動でのセグメンテーションと隠れた領域の「幻覚的」描画（オクルージョン補完）が必要だったが、本手法はこれを自動化する。

### 主な貢献

1. **単一画像 → 2.5D キャラクター変換**: 完全に塗りつぶされた（inpainted）セマンティック RGBA ボディパーツレイヤーとオクルージョン補完、描画順序推定を生成
2. **データエンジン**: Live2D モデルから弱い教師信号をブートストラップし、19クラスの身体部位アノテーション・オクルージョン領域・フラグメントレベルの描画順序を生成
3. **Body Part Consistency Module（BPCM）**: 拡散ベースの2段階学習戦略で、パーツ間の完全性とクロスレイヤー整合性を向上

## 1. はじめに（Introduction）

アニメコミュニティは世界中の視聴者に向けて大量のコンテンツを制作している。近年、高品質な静止イラスト（*Tachi-e*）を「モーションイラスト」に変換するトレンドが急成長しており、VTubing、2Dゲーム、ビジュアルノベルのビジュアルコアとして活用されている。

しかし、この変換は容易ではない。課題は以下の通り：

- 高品質なアニメーションとインタラクティビティの実現
- オリジナルの美的整合性の厳密な保持
- フル3Dモデリングパイプラインは、アニメアートワークには非実用的かつ美的に不適合

### 業界標準：Live2D による 2.5D アプローチ

業界ではこの課題に対し、**Live2D** などの「2.5D」アプローチを採用している：

1. オリジナルのフラットイラストをセマンティックに意味のある2Dレイヤーに分割
2. 動きに必要な隠れた領域（オクルージョン）を追加描画
3. モーションとデフォメーションタイミングでレイヤーを操作 → 3Dボリュームの錯覚を生成

**問題点**: このワークフローはクリエイターに膨大な負担を強いる：

- 単一イラストを数十〜数百のレイヤーに手動分離
- 正確な配置順序の明示的決定
- 全レイヤーの隠れた領域をもっともらしいビジュアルで「幻覚的に」補完 → 単調で技術的に困難

### 本研究のアプローチ

本研究は、**単一の静止イラスト**から、完全に分離・塗りつぶされた 2.5D 対応キャラクターモデルへの自動変換を実現する：

- 画像を **4チャンネル**（RGBA）の意味的に異なるボディパーツレイヤーに分解
- 細粒度の**描画順序**を推論
- グラウンドトゥルースアノテーションが存在しないため、2Dセマンティックセグメンテーションからラベルをブートストラップする**スケーラブルなデータエンジン**を構築

## 2. 関連研究（Related Work）

### 2.1 アニメキャラクターのセマンティックパーシング

- 初期は対話的なユーザー入力に依存 → 現在は完全自動のデータドリブンアーキテクチャへ移行
- ボディ構造のキャプチャにはキーポイント推定 [6, 31, 36] を活用
- SAM シリーズ [3, 23] は汎用性が高いが、アニメ画像とのドメインギャップが存在
- **根本的制約**: 既存手法は全て2D平面パーシングに限定され、隠れた解剖学の推論（"see-through"能力）を持たない

### 2.2 画像レイヤー分解と Z-Order 推論

- 最近の拡散ベース手法 [43, 51, 54] は4チャンネル RGBA レイヤーの生成・分解を可能にした
- しかし、固定的なトップダウンのレイヤー順序を前提とし、髪がフェイスレイヤーを「サンドイッチ」するようなアニメ特有の複雑な重層化に対応できない
- アモーダル補完やシーンオブジェクトデオクルージョン [21, 26] は粒度が粗く、グローバル整合性が犠牲

### 2.3 アニメキャラクターのアニメーション化

| アプローチ | 手法例 | 課題 |
|:---|:---|:---|
| ARAP 変形 [17] | 手動レイヤーワーピング | レイヤリングとアノテーションに依存 |
| 疑似3Dプロキシ [37, 49] | キャラクターシート | 手動依存 |
| Live2D 自動化 [12, 13] | テキスト → Live2D | テンプレートメッシュへのテクスチャマッピング |
| 3D リフティング [7, 9, 32] | メッシュ生成 | フラットシェーディングと非ユークリッド幾何との衝突 |
| 動画拡散 [11, 16, 48, 52] | スタイル化アニメ動画 | 解像度・時間整合性の限界、リアルタイム不可 |

## 3. データセット（Dataset）

### 3.1 2D セマンティックセグメンテーション データエンジン

#### 3.1.1 Live2D データ構造

Live2D をデータソースに選択した理由：

- **業界標準**であり、豊富なモデルが公開されている
- 各モデルは *ArtMesh* の集合体 — テッセレーション三角メッシュで、テクスチャアトラスの局所領域にマッピング
- レンダリング時にオリジナル画像をメッシュにマッピングし、描画順序インデックスで離散的な z-buffer 信号を提供
- **ジオメトリベースのビジビリティマスク**から、各 ArtMesh のピクセルパーフェクトな境界・表示/非表示・ブレンディング情報を抽出可能

#### 3.1.2 2D アニメボディセマンティクスのアノテーション

**19クラスの身体部位タクソノミー**を定義：

| カテゴリ | 部位 |
|:---|:---|
| 頭部 | hair（髪）, headwear（帽子等）, face（顔）, eyes（目）, eyewear（眼鏡等）, ears（耳）, earwear（イヤリング等）, nose（鼻）, mouth（口）, neck（首） |
| 上半身 | neckwear（ネクタイ等）, topwear（上着）, handwear（手袋等） |
| 下半身 | bottomwear（ズボン/スカート）, legwear（靴下/タイツ）, footwear（靴） |
| その他 | tail（しっぽ）, wings（翼）, objects（小物） |

自動ラベリング手順：

1. **Danbooru タグ + wd-eva02-large-tagger-v3** [1, 39] による画像タグ予測
2. **GradCAM++** [4] でクラスごとのアクティベーションマップを計算
3. **ArtMesh ベースの投票**: 各フラグメントのビジビリティマスク内でアクティベーションスコアを空間平均し、最大スコアのクラスに割り当て → ノイズ除去・境界整合

#### 3.1.3 Multi-Decoder SAM による反復的精緻化

GradCAM の初期結果では曖昧さが残る（ポニーテールの誤分類、肩紐とリボンの混同など）。

**解決策**: SAM-HQ [20] のセグメンテーション事前知識を活用

- SAM のプロンプタブルデコーダを **19個の独立したマスクデコーダ**に置換（各クラス専用）
- プロンプト埋め込みをゼロにして、画像埋め込みから自律的にクラス固有特徴を学習
- **自己改善学習ループ**: 初期の GradCAM ラベル → モデル予測 → 新しいラベルで教師信号を更新 → 反復的に精度向上

### 3.2 2.5D レイヤリングアノテーション

2D セグメンテーションで予測したピクセルレベルラベルを、ArtMesh に投影して 2.5D データセットを構築：

- 表示フラグメント: ピクセルの多数決で直接クラス割り当て
- 隠れたフラグメント: **階層的伝播戦略**で推論
  - **Semantic String Matching（文字列マッチング）**: "PartHair" などの名前が既知のクラスと一致するか
  - **Sibling Voting（兄弟投票）**: 同一階層レベルの ArtMesh で多数決
  - **Recursive Parent Voting（再帰的親投票）**: 親ノードを再帰的に遡って多数決

## 4. 手法（Methodology）

### 4.1 セマンティックレイヤー分解

入力画像 *I* ∈ ℝ^(H×W×3) を、N=19 のセマンティックレイヤー L = {L₁, ..., L_N} に分解する。各レイヤー L_k ∈ ℝ^(H×W×4) は完全に塗りつぶされた RGBA 画像で、可視面と隠れたオクルージョン領域の両方をキャプチャする。

#### 4.1.1 透明度デコーダ（Transparency Decoder）

SDXL [34] の VAE エンコーダ ε_x とデコーダ D_x はそのまま維持し、新たに**透明度デコーダ D_a** を追加：

- ボディパーツの RGB チャンネルを潜在変数 z_k = ε_x(L_k^(c)) にエンコード
- D_x(z_k) で RGB を復元
- D_a は z_k と L̂_k^(c) の両方を条件として、パディングされたガウシアン P̂_k とアルファチャンネル L̂_k^(a) を予測

**損失関数**:
```
L_dec = ||L_k^(c) - L̂_k^(c)||₂ + ||L_k^(a) - L̂_k^(a)||₂ + λ_disc · L_disc
```
ここで L_disc は PatchGAN 判別器の損失（λ_disc = 0.01）

#### 4.1.2 ボディパーツ分解のための拡散モデル

**2段階の local-to-global 学習戦略**を採用：

**第1段階: 単一ボディパーツの抽出**

- バニラ SDXL をファインチューンし、1つのターゲットパーツを高忠実度で抽出
- 入力画像 I を画像条件エンコーダ τ_I(I) でゼロ畳み込み経由でエンコーダに注入
- セマンティッククラス c_k を離散埋め込み τ_C(c_k) でクロスアテンション経由で注入
- ε-prediction で学習:

```
L_stage1 = E[||ε - ε_θ(z_k^(t), t; τ_I(I), τ_C(c_k))||₂²]
```

**第2段階: 整合的なボディパーツ分解**

第1段階は個別パーツの品質は高いが、全パーツを一貫して分解する際に以下の問題が発生：

- 一部のセマンティックレイヤーがほぼ空になる
- 視覚的に類似した隣接領域を誤って別パーツに割り当てる（例: 上半身の服 → 袖と混同）

**解決策: Body Part Consistency Module（BPCM）**

- 全 N 個のパーツ潜在変数をパーツ次元 c に沿ってスタック → 動画拡散の時間次元に類似
- U-Net の各空間アテンションブロックの後に BPCM を挿入
- パーツ次元 c に沿ってアテンションを実行 → 各レイヤーが他の全レイヤーの予測を参照可能
- c 方向の畳み込みは導入しない（セマンティックレイヤーは大部分が独立）
- パースライスのクラス埋め込み τ_C(c_k) が**局所的セマンティック制約**として機能

```
L_stage2 = E[||ε - ε_φ(Z^(t), t; τ_I(I), τ_C(C))||₂²]
```

### 4.2 分解されたレイヤーの再構成

#### 4.2.1 擬似深度（Pseudo-Depth）による描画順序推論

複雑なオクルージョンの中で、シーン整合的な描画順序を推論するため、各セマンティックレイヤーごとに**ピクセルレベルの擬似深度マップ D_k** を予測する。

**バックボーン**: **Marigold** [19]（最先端のアフィン不変深度推定器）

- Marigold は写実的画像向けだが、相対的な深度推定における強力な幾何学的事前知識がアニメコンポジティングの描画順序推論にも有効
- 擬似深度は実測3D深度ではなく、描画順序を表現

**2段階アプローチ**:

1. **Stage 1（パーツ別擬似深度）**: タグ c_k で条件付けして、特定セマンティック領域の擬似深度マップを予測 → アニメパーツの特殊なトポロジー（顔に対する髪の曲率など）に適応
2. **Stage 2（グローバル整合深度）**: 19枚の深度マップをスタックし、BPCM を再利用 → パーツ間の統一的・グローバルに順序付けされた擬似深度階層を確立

#### 4.2.2 深度ガイド付きレイヤー層化

RGBA レイヤーと擬似深度から、z-buffering で合成が可能。しかし：

- 単純に各パーツに代表深度値を割り当てて z-sort する方法では、**クラス内インターリーブ**を表現できない
- 例: 髪が顔の前面と背面の両方に存在する「サンドイッチ」ケース

**解決策**: セマンティックレイヤーを少数の深度ストラタに細分化

- 予測された擬似深度がセルフオクルージョンパーツ内で分離モードを形成する傾向を利用
  - 例: 前髪 vs 後ろ髪
- **K-Means クラスタリング**（K=2）でアルファマスク内の擬似深度値をクラスタリング → 前面/背面に分割
- 適用対象: **Hair, Handwear, Topwear, Bottomwear**
- 分割で新たに露出した領域は **LaMa** [42]（汎用インペインター）で補完

## 5. 実験（Experiments）

### 5.1 システム評価

#### 5.1.1 レイヤー分解と再構成

入力2Dイラスト1枚から、予測レイヤーと擬似深度により入力キャラクターのほぼ完璧な再構成を実現。隠れた部分を「幻覚的に」生成し、大きなデフォメーションにも対応可能。

**比較対象**: Qwen-Image-Layered [51]

- Qwen は特定のボディパーツの正確な抽出に苦戦
- 異なるセマンティックレイヤーの結合や、単一パーツの不整合な分断が発生
- 復元損失は低いが中間レイヤーが非整合でアニメーションに使用不可

### 定量評価比較

![定量評価: See-through vs ベースライン](stock/Papers/SeeThrough_AnimeLayerDecomposition/diagrams/diagram_evaluation.png)

*図3: ベースライン（SAM+LaMa）との定量比較。提案手法は LPIPS 46.2%改善、PSNR +6dB、Mask MSE 65.3%改善、FID 77.4%改善を達成。BPCM による擬似深度推論の AbsRel も 39.1% 改善*

#### 5.1.2 2D セマンティックボディパーシング

SAM3 [3] との比較：

- SAM3 はアニメキャラクターで不完全・曖昧なセグメンテーションを頻出
- 欠落領域（髪やズボン）やパーツ間のマスク重複が典型的な失敗パターン
- SAM3 は "see-through" 能力を持たない純粋な2D手法

**定量評価（Table 1）**:

| 手法 | LPIPS ↓ | PSNR ↑ | SSIM ↑ | Mask Dice loss ↓ | Mask MSE ↓ | FID ↓ |
|:---|:---|:---|:---|:---|:---|:---|
| SAM+LaMa | 0.2880 | 12.2802 | 0.8445 | 0.4336 | 0.1020 | 81.1419 |
| Ours w/o BPCM | 0.1952 | 16.2350 | 0.9053 | 0.6480 | 0.0640 | **16.7069** |
| **Ours Full** | **0.1549** | **18.2965** | **0.9230** | **0.3855** | **0.0354** | 18.3700 |

#### 5.1.3 Body Part Consistency Module のアブレーション

BPCM の効果：

- **(i) レイヤー分解**: 再構成メトリクスが改善 → パーツの完全性とクロスレイヤー整合性が向上
- **(ii) 描画順序推論**: 擬似深度推定の品質も向上
  - AbsRel（平均絶対相対誤差）: 0.1549 → 0.0943
  - δ₁ スコア: 0.8427 → 0.9103
- FID のわずかな悪化は、グローバル整合性の強制が出力多様性を低下させるため（予想通り）

### 5.2 アプリケーション

#### パペットアニメーション

- 単一イラストから安定した2.5Dアニメーション
- 確定的な2Dデフォメーションとコンポジティング + リアルタイム物理（バネベースの髪ダイナミクス）
- **Animated Drawings** [40] との比較: 本手法は複雑なデフォメーションでもオクルージョンとレイヤー関係を維持
  - スカート—靴—髪の層化が保持される
  - Animated Drawings で見られるティアリング/ストレッチングアーティファクトを回避
- **MikuDance** [52] との比較: MikuDance は時間的不整合と局所歪みが発生し、リアルタイム性能も未達

#### トーキングヘッド VTubing

- 顔と髪のレイヤーを利用したリアルタイム「トーキングヘッド」システム
- 分離された RGBA パーツにより、2D デフォーマーで表情と頭部動作の安定制御
- 細かいアニメディテール（線画、フラットシェーディング）を保持しつつ、大きな顔面デフォメーションに対応
- 3D フェイシャルランドマークで駆動

#### プロフェッショナルへの評価

- レイヤード PSD ファイルとしてエクスポートし、プロのアニメーターにキーフレーム作成を依頼
- 出力は「ほぼプロダクションレディ」で、わずかな手動修正のみ必要との評価

## 6. 結論（Conclusion）

本研究では、単一のアニメイラストから、完全なセマンティック RGBA ボディパーツレイヤーへの分解と、層化された描画順序の予測により、2.5D 対応キャラクターモデルを生成するフレームワークを提示した。

**成果**:
- ブートストラップされたデータセットとクロスパーツ分解モデルにより、美的に忠実な分解を実現
- ほぼ完璧な再構成を可能にし、実用的な下流アニメーションワークフローをサポート

**制限事項**:
- 予測レイヤー間のわずかな重複が、ボディ外部の領域で発生することがある（レイヤーエディタで容易に修正可能）
- 深度ガイド付き層化は、擬似深度が信頼性の高い離散的境界を常に誘導するわけではなく、安定した複数サブレイヤーへの分割が困難な場合がある

**今後の展望**: 自動アニメーションタイミングの予測を含む、分解の更なる改善
