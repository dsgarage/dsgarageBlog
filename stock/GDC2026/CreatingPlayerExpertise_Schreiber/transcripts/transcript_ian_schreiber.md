## キーコンセプト整理

### ワーキングメモリ（約7スロット）
- タスクの難しさ = ワーキングメモリのスロット消費量
- PCのRAM + CPUに相当
- チャンキング: 既知パターンでまとめることでスロット節約

### 3種類の認知負荷
1. **Intrinsic Load**（内在的負荷）: 扱うデータそのもの（例: X=Jump）
2. **Extraneous Load**（外在的負荷）: データ処理に必要な余計な負荷（例: どのXボタン？）→ **極力減らす**
3. **Germane Load**（学習的負荷）: 戦略・判断に使う負荷 → **適切にコントロール** = フロー

### 長期記憶化
- Germane Loadによる練習 → スキルが長期記憶に転送
- 長期記憶に入ったスキルはワーキングメモリを消費しない
- = スロットが空く = より高度な判断が可能 = **Expertise**

### 認知過負荷の悪循環
1. Germane Load が多すぎる → スロット超過
2. 脳のオーバークロック（一時的にスロット増加）→ ストレス・不安
3. 短時間: パニック（許容範囲）
4. 長時間: 認知疲労 → スロット数**減少**（デバフ）→ さらに悪化 → スパイラル
5. 最終的に闘争・逃走モード → デザインとしては**失敗**

### Sid Meierの引用
- 「ゲームとは興味深い決断の連続である」
- Germane Load = 興味深い決断を生むのに必要な認知負荷
- 適度な Germane Load = フロー状態

### プレイテストでの認知過負荷の兆候
- 同じミスを繰り返す
- コントロールやメカニクスを忘れる
- UIと「戦っている」
- 無力感・「上手くなれない」という感覚
- ゲームから弾かれる（「分からない、分かりたくもない」）
- レイジクイット

### 実践的アドバイス
1. **ワーキングメモリスロットを数えろ** — 実際に座ってプレイヤーが必要とするデータスロット数をカウントする
2. **プレイヤーの前提知識を考慮** — 「Haloコントロール」で1スロットの人もいれば、初めてのゲームで12コントロール全部が個別スロットの人もいる
3. **不要な情報を早く与えすぎるな** — 「しゃがみを教えたのに10時間使わない」FPS問題。ボードゲームの「アクションポイント3点/ターン」を先に言って、何に使えるかは3ページ後に説明する問題
4. **Simplification Pass を行え** — デザイナーは設計中に複雑さを追加しがち。武器と防具で別画面が本当に必要か？グレネードを武器として扱えないか？
5. **追加した複雑さの正当性を検証** — コントロールが固まったら、追加の認知負荷がゲーム体験に見合うか厳しくテスト
6. **特に注意すべきジャンル** — 教育ゲーム/シリアスゲーム、ジャンル入門作、ジャンルハイブリッド（半分のメカニクスしか知らないプレイヤーが来る）
