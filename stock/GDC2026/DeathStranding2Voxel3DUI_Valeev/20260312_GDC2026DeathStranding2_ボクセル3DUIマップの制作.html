‘DEATH STRANDING 2’: Making of Voxel 3D UI Map

セッション情報

項目 内容
スピーカー Ildar Valeev (KOJIMA PRODUCTIONS)
トラック Game & Production Technology, Visual Development
形式 Lecture
日時 2026年3月12日（木）11:50–12:20
場所 Room 2001, West Hall

セッション概要

KOJIMA PRODUCTIONS の Ildar Valeev 氏が、Death Stranding 2 におけるボクセルベース 3D UI マップの制作過程を解説したセッション。ゲーム内のマップ表示にボクセル技術を採用した理由、データパイプラインの設計、レンダリング最適化、そして開発哲学について約30分間にわたって語られた。

セッション開始前の会場

タイトルスライド

セッション要約

1. 導入 — マップの重要性とUIの課題（11:49〜11:52）

  • Death Stranding 2 におけるマップはゲームの中核的要素
  • UIマップとして独自の課題がある — 広大なオープンワールドをどうUIとして表現するか
  • ボクセル表現を採用した理由と背景の説明

2. データ処理パイプライン（11:52〜11:56）

  • プレイヤーの意思決定をサポートするマップ設計思想
  • 解像度・フォーマットの検討 — 低解像度化の方針を選択
  • ポイントクラウドデータの取得とキャプチャ手法
  • カメラと重力を使ったスライス処理
  • ボクセルの格納方式 — 各グレイン（粒度）の保持方法
  • 位置情報のブレンディング — 両腕の平均位置を使う手法

3. ボクセル生成とプロパティ（11:56〜12:00）

  • ゲーム内データを直接活用する方針
  • 独自のレイキャスターの自作
  • 各ボクセルが保持する属性 — パワー（power）、強度（intensity）、明度（loudness）
  • データクリーニングとリスク管理のプロセス
  • 3日間で大量のデータを処理する課題への対応

4. レンダリングと最適化（12:00〜12:04）

  • 描画パイプラインのワークフロー設計
  • 絶対値（absolute value）をユニバーサルに持たせる設計
  • フェーズを重ねる反復的処理パイプライン
  • ボクセルデータのキーデータに基づく処理
  • ドリルスコアを使ったジェネレータ
  • レベニューロス（データ損失）のサマリー

5. メモリ管理とLOD制御（12:04〜12:08）

  • ボクセルデータはギガバイト規模のバイナリデータ
  • メモリと処理の制御が不可欠
  • 複数ステップに分けたファイル処理パイプライン
  • プレイヤーに不可視のオブジェクト処理
  • ノイズ原理を使ったLOD/簡略化手法
  • 可視領域の判定と描画対象の取捨選択

6. ピクセルシェーダーと深度補正（12:08〜12:10）

  • 自己遮蔽（self-occluding）処理
  • ピクセルシェーダーで深度値を設定し、エネルギー影響を補正
  • シンプルな「チート」テクニックによる最適化
  • アンプリチュード（amplitude）によるカラー/強度マッピング

7. 開発哲学と締めくくり（12:10〜12:13）

  • アニメーション・配置など様々な用途への応用
  • テクニカルパートの結論 — 開発プロセスの振り返り
  • 容易に組み込める（easily embedded）ことの重要性
  • クラシックマテリアル手法ではなく独自プロセスを選択
  • デスプロジェクション（描画投影）の強化

8. Q&A（12:13〜12:15）

  • プレゼン本編終了後、聴衆からの質疑応答
  • カメラ・音楽・画像の活用に関する質問

技術的キーワード

  • Voxel（ボクセル）
  • Point Cloud（ポイントクラウド）
  • Ray Caster（レイキャスター）
  • Pixel Shader / Depth Value（ピクセルシェーダー / 深度値）
  • LOD（Level of Detail）
  • Amplitude Mapping（振幅マッピング）
  • Self-Occlusion（自己遮蔽）
  • Binary Data / Memory Management（バイナリデータ / メモリ管理）

所感

  • 音声認識の精度が低く、専門用語が正確に拾えていない箇所が多い。スライド写真との照合が必要
  • ボクセルをUIマップに採用するという発想自体がユニーク。通常のミニマップとは全く異なるアプローチ
  • GB級のバイナリデータをリアルタイムで扱うための最適化手法が印象的
  • 独自レイキャスターの自作や深度補正のチートテクニックなど、実践的なノウハウが多い
