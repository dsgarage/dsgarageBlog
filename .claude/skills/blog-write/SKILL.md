# ブログ記事執筆スキル

## 使い方

ユーザーが「記事を書いて」「ブログ書いて」「<テーマ>について執筆して」等と言ったらこのスキルを使う。

## 手順

### 1. テンプレート読み込み

```bash
cat TechBlog/_template.md
```

テンプレートの構成とフロントマターの書式を確認する。

### 2. 執筆ガイドライン確認

```bash
cat TechBlog/BLOG_WRITING_GUIDELINE.md
```

品質基準・リード文の3要素・表記ルールを確認する。

### 3. 原稿ファイル作成

ファイル名: `TechBlog/YYYYMMDD_タイトル.md`（今日の日付を使用）

以下の構成で作成:
1. **フロントマター** — title, tags, platforms (qiita/zenn/note)
2. **リード文** — 課題提起・解決策・読後の得られるもの の3要素
3. **本文** — 見出し h2/h3 で構造化、コード例・図解指示を含む
4. **まとめ** — 要点箇条書き + 次のアクション
5. **UniMCP4CC 誘導** — 記事末尾に含める（ガイドライン参照）

### 4. 図解が必要な場合

`TechBlog/images/` に `.dot` ファイルを作成し、ビルドスクリプトで PNG 化:

```bash
cd TechBlog && ./build-diagram.sh
```

### 5. 確認

- 製品名表記: **UniMCP4CC** が正式名称
- 画像パスが `images/` を指していること
- フロントマターが正しいこと

## 注意事項

- `docs/` には直接ファイルを置かない
- 調査素材は `stock/<topic>/` に置く（記事原稿とは分離）
