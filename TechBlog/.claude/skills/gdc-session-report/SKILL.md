# GDC セッションレポート執筆スキル

セッション「$ARGUMENTS」の GDC レポート記事を執筆する。

---

## 前提条件

以下の素材がセッションフォルダに格納されていること:

```
../stock/GDC20XX/<SessionName>/
├── transcripts/          # Talksribe 録音テキスト（.txt）
├── slides/               # スライド写真（.jpg/.png）
└── （その他参考資料）
```

## Step 0: 素材の確認

1. セッションフォルダを特定（`$ARGUMENTS` から検索）
2. `transcripts/` 内のテキストファイルを全て読む
3. `slides/` 内の画像ファイルを全て読む（Read ツールで画像を視覚的に確認）
4. 素材が不足していればユーザーに報告

## Step 1: 記事構造の設計

トランスクリプトとスライドを通読し、以下を特定する:

- セッションの主題と技術領域
- スピーカーの所属・肩書き
- 使用エンジン（Unreal / Unity / 自社エンジン等）
- 主要な技術トピック（H2 セクションの候補）
- 各スライドがどのトピックに対応するか

### 各セクションの品質基準

1. **スライドのテキストは全文書き起こし**: ラベル、コード、注釈、箇条書きの全てを転記し解説
2. **スピーカーの発言を引用**: トランスクリプトから重要な発言を `>` で引用
3. **専門用語は初出で解説**: 括弧内に日本語説明（例: SSR（Screen Space Reflections：画面上の...））
4. **数値は表で提示**: 改善量、コスト、解像度などは Markdown 表にまとめる
5. **コードスニペットは言語指定**: ```hlsl, ```cpp, ```csharp 等

### Unity ガイドの品質基準（※非 Unity セッションの場合）

1. **Unity API は公式リファレンスで検証**: クラス名、メソッド名、名前空間を正確に
2. **問題・課題を具体的に記述**: 「Unity で同様の問題が起きるケース」を 3〜5 個列挙
3. **コード例は実行可能なレベル**: using 文は省略可だが、クラス・メソッド名は正確に
4. **Forward+ のライト上限など仕様値は最新を確認**: URP Config Package 等のバージョン依存に注意

## Step 3: 図解の作成（draw.io）

以下の基準で図解が有効なセクションを特定し、draw.io XML で作成する:

| 図解が有効なケース | 例 |
|:---|:---|
| データ構造の比較（Before/After） | リンクリスト vs ビット配列 |
| パイプラインのフロー | レンダリングパス、最適化サイクル |
| 時系列の動作（フレーム単位） | リングバッファ、ダブルバッファ |
| 概念の対比 | Raster vs Compute、ベイクド vs ダイナミック |

### 図の作成手順

1. `mcp__drawio__open_drawio_xml` で draw.io エディタに図を表示
2. `.drawio` ファイルを `slides/` に保存（後から編集可能にする）
3. `drawio --export --format png --scale 2` で PNG エクスポート
4. 記事に `![説明](../stock/GDC20XX/.../slides/diagram_xxx.png)` で埋め込み
5. イタリック体のキャプションを付ける

### 図のスタイルガイド

- 赤系（`#f8cecc`）= 問題・非効率・アンチパターン
- 緑系（`#d5e8d4`）= 解決策・改善・推奨
- 青系（`#dae8fc`）= 処理・データ・ニュートラル
- 黄系（`#fff2cc`）= 入力・キー要素・注目ポイント
- 紫系（`#e1d5e7`）= 特殊処理・Wave操作・統合

## Step 5: ユーザーレビュー

PDF をユーザーに提示し、以下の観点でフィードバックを求める:

- スライドの書き起こしが正確か
- 解説がわかりやすいか
- Unity ガイドの API が正確か
- 追加の図解が必要か

フィードバックを反映し、PDF を再生成する。

## 注意事項

- **UniMCP4CC 誘導は不要**（GDC 記事には含めない）
- **事実ベースを維持**: トランスクリプトにない情報を創作しない
- **英語引用は原文を残す**: 重要な発言は英語原文 + 日本語意訳
- **目標行数**: 400〜1100 行（セッション長に応じて調整。30分→600行目安、60分→1000行目安）
