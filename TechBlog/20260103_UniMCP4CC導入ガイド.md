# UniMCP4CC導入ガイド

「MCPでUnityを操作できる」——前回、その可能性を解説しました。
しかし、実際に使うにはセットアップが必要です。

「設定が複雑で挫折した」という声をよく聞きます。
Package Managerからインストール、MCP Bridgeの設定、Claude Codeへの登録……。
どこから手を付ければいいか分からなくなります。

本記事では、**UniMCP4CCを5分でセットアップする手順**をステップバイステップで解説します。
この手順通りに進めれば、確実に動作する状態まで到達できます。

:::message
**UniMCP4CCとは**

**UniMCP4CC**（Unity MCP Server for Claude Code）は、Claude CodeからUnity Editorを操作できるようにするパッケージです。

- GitHub: [dsgarage/UniMCP4CC](https://github.com/dsgarage/UniMCP4CC)
- 対応Unity: 2021.3 LTS以降
- ライセンス: MIT
:::

## 必要な環境

### 前提条件

以下の環境が必要です：

| 項目 | 要件 |
|:---|:---|
| **Unity** | 2021.3 LTS 以降 |
| **Node.js** | v18 以上 |
| **Claude Code** | インストール済み |
| **OS** | Windows / macOS / Linux |

### 事前確認

ターミナルで以下のコマンドを実行し、環境を確認してください：

```bash
# Node.jsのバージョン確認
node --version
# → v18.0.0 以上であればOK

# Claude Codeの確認
claude --version
# → バージョンが表示されればOK
```

Node.jsがインストールされていない場合は、[nodejs.org](https://nodejs.org/)からダウンロードしてください。

## セットアップ手順

### Step 1: Unityパッケージのインストール

#### 1-1. Package Managerを開く

Unity Editorのメニューから開きます：

```
Window > Package Manager
```

#### 1-2. Git URLからパッケージを追加

Package Managerの左上にある「+」ボタンをクリックします：

```
+ ボタン > Add package from git URL...
```

#### 1-3. URLを入力

以下のURLを入力し、Enterを押します：

```
https://github.com/dsgarage/UniMCP4CC.git
```

パッケージがインポートされるまで待ちます（通常30秒〜1分程度）。

#### 1-4. インストール確認

メニューに以下が追加されていれば、インストール成功です：

```
Tools > UniMCP4CC > ...
```

---

### Step 2: MCP Bridgeの準備

#### 2-1. Bridgeフォルダを確認

パッケージをインストールすると、以下の場所にMCP Bridgeが含まれています：

```
Packages/com.dsgarage.unimcp4cc/mcp-bridge/
```

#### 2-2. 依存関係のインストール

ターミナルでBridgeフォルダに移動し、依存関係をインストールします：

```bash
cd /path/to/your/unity-project/Packages/com.dsgarage.unimcp4cc/mcp-bridge
npm install
```

`npm install`が完了するまで待ちます（通常1〜2分程度）。

---

### Step 3: Claude Code側の設定

#### 3-1. MCPサーバーを登録

ターミナルで以下のコマンドを実行します：

```bash
claude mcp add unity-local -- npx ts-node /path/to/mcp-bridge/index.ts
```

**重要**: `/path/to/mcp-bridge/` は実際のパスに置き換えてください。

パスの例：
```bash
# macOS
/Users/username/UnityProjects/MyGame/Packages/com.dsgarage.unimcp4cc/mcp-bridge/index.ts

# Windows
C:\Users\username\UnityProjects\MyGame\Packages\com.dsgarage.unimcp4cc\mcp-bridge\index.ts
```

#### 3-2. 登録確認

以下のコマンドで登録を確認します：

```bash
claude mcp list
```

`unity-local` が表示されれば登録成功です。

---

### Step 4: 接続確認

#### 4-1. Unity Editorを起動

Unityプロジェクトを開きます。Playモードにする**必要はありません**。

#### 4-2. MCP Serverを起動

Unityのメニューから起動します：

```
Tools > UniMCP4CC > Start Server
```

Consoleに以下のメッセージが表示されれば起動成功です：

```
[MCP] Server started on port 5052
```

#### 4-3. Claude Codeで確認

ターミナルでプロジェクトフォルダに移動し、Claude Codeを起動します：

```bash
cd /path/to/your/unity-project
claude
```

Claude Codeで以下を入力します：

```
Hierarchyにあるオブジェクトを一覧表示して
```

Scene内のオブジェクト一覧が返ってくれば、**接続成功**です。

## うまくいかない時のチェックリスト

### 症状別の対処法

#### 「MCP Serverが起動しない」

**チェック項目：**
- Unity Editorは起動しているか
- Tools > UniMCP4CC > Start Server を実行したか
- Consoleにエラーが出ていないか

**対処法：**
1. Unity Editorを再起動
2. Tools > UniMCP4CC > Stop Server → Start Server
3. パッケージを再インポート

#### 「Claude Codeから接続できない」

**チェック項目：**
- `claude mcp list` で unity-local が表示されるか
- パスは正しいか（スペースや日本語を含んでいないか）
- Node.jsのバージョンは v18 以上か

**対処法：**
```bash
# MCPサーバーを削除して再登録
claude mcp remove unity-local
claude mcp add unity-local -- npx ts-node /correct/path/to/mcp-bridge/index.ts
```

#### 「操作が反映されない」

**チェック項目：**
- Unity EditorでSceneビューを確認しているか
- Hierarchyは正しいSceneを表示しているか
- GameObjectがActive状態か

**対処法：**
1. Sceneビューを確認
2. Hierarchyをリフレッシュ（F5キー）
3. 別のSceneを開いていないか確認

#### 「エラーが返ってくる」

**よくあるエラーと原因：**

| エラー | 原因 |
|:---|:---|
| "GameObject not found" | オブジェクト名が間違っている |
| "Component already exists" | すでに追加済み |
| "Invalid path" | パスの形式が不正 |

## 設定ファイル

### Unity側の設定

設定は `ProjectSettings/UniMCP4CCSettings.asset` に保存されます。

| 設定項目 | 説明 |
|:---|:---|
| Server Port | 使用するポート番号（デフォルト: 5052） |
| Auto Start | エディタ起動時に自動開始するか |
| Log Level | デバッグ情報の出力レベル |

### Claude Code側の設定

登録情報は `~/.claude.json` に保存されます：

```json
{
  "mcpServers": {
    "unity-local": {
      "command": "npx",
      "args": ["ts-node", "/path/to/mcp-bridge/index.ts"]
    }
  }
}
```

## ポート競合の解決

デフォルトポート（5052）が他のアプリケーションで使用されている場合、ポートを変更する必要があります。

### 1. 使用中のプロセスを確認

```bash
# macOS / Linux
lsof -i :5052

# Windows
netstat -ano | findstr :5052
```

### 2. ポートを変更

**Unity側：**
```
Tools > UniMCP4CC > Settings > Port: 5053
```

**MCP Bridge側：**
```bash
# 環境変数で指定
UNITY_MCP_PORT=5053 npx ts-node /path/to/mcp-bridge/index.ts
```

## よくある質問

### Q: Play中でないと動かない？

**A: Playモードでなくても動作します。**

Edit Mode（再生していない状態）で操作可能です。むしろ、Scene構築はEdit Modeで行うことが多いです。

### Q: 複数のUnityプロジェクトで使える？

**A: はい。**

ただし、同時に複数のMCP Serverを起動する場合は、ポートを分ける必要があります。

### Q: ネットワーク越しに使える？

**A: 可能ですが、セキュリティに注意してください。**

デフォルトはlocalhostのみです。外部公開する場合は認証を追加してください。

### Q: 壊れたSceneを復元できる？

**A: Undo機能があります。**

```
「最後の操作を取り消して」
```

または `Ctrl+Z` / `Cmd+Z` で通常のUndoが使えます。

## まとめ

### セットアップ手順

```
1. UniMCP4CCをPackage Managerからインストール
2. mcp-bridgeで npm install
3. claude mcp add でサーバー登録
4. Unity側で Start Server
5. Claude Codeから接続確認
```

### 所要時間

- 慣れれば**5分**で完了
- 初回は**10〜15分**程度

### トラブル時は

1. Consoleのエラーメッセージを確認
2. パスに日本語やスペースが含まれていないか確認
3. ポート競合がないか確認
4. 解決しない場合は[GitHub Issues](https://github.com/dsgarage/UniMCP4CC/issues)へ
