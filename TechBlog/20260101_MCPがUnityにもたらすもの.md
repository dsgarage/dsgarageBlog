# MCPがUnityにもたらすもの

「手動作業5分を、なんとか短縮できないか」——前回、Unity開発における壁を解説しました。
AIがコードを生成しても、Scene構築やコンポーネント設定には手作業が必要でした。

しかし今、その壁を突破する技術があります。
**MCP（Model Context Protocol）**を使えば、自然言語でUnity Editorを操作できます。

```
あなた: 「Playerを作成して、Rigidbodyを追加して、位置を(0, 1, 0)にして」
Claude: 「完了しました」
```

本記事では、MCPの仕組みと、それがUnity開発にもたらす具体的な変化を解説します。
350以上のAPIで何ができるのかを理解することで、新しい開発ワークフローを設計できるようになります。

## MCP（Model Context Protocol）とは

### 一言で言うと

**MCP**は、Anthropic社が策定した**AIとツールを接続するためのオープンプロトコル**です。

### わかりやすく言うと

MCPは「AIが外部ツールを操作するための共通言語」です。

```
MCPの役割:
├── AIが外部ツールを操作するための「共通言語」
├── USBのような標準規格
└── AIと世界をつなぐ橋
```

USBがどんなメーカーのデバイスでも接続できるように、MCPはどんなツールでもAIと接続できます。

### なぜMCPが重要なのか

MCPがない世界では、ツールごとに個別の連携実装が必要でした。
MCPがあることで、一度の実装でどのAIからでもツールを利用できます。

```
MCPがない世界:
├── ツールごとに個別の連携が必要
├── 開発者が毎回実装する
└── 非効率で脆弱

MCPがある世界:
├── 標準プロトコルで接続
├── 一度実装すれば動く
└── 安定して拡張可能
```

## 自然言語でUnityを操作する

### 従来の操作（GUI）

従来、GameObjectを作成してコンポーネントを設定するには、以下の手順が必要でした：

```
1. Hierarchyで右クリック
2. Create Empty を選択
3. 名前を「Player」に変更
4. Inspectorでpositionを設定
5. Add Component → Rigidbody
6. mass を 2 に設定
7. Add Component → Capsule Collider
8. height を 1.8 に設定

所要時間: 2〜3分
```

### MCPを使った操作

MCPを使うと、同じ操作が自然言語で完了します：

```
あなた: 「Playerを作成して、Rigidbodyとカプセルコライダーを追加、
        massは2、高さは1.8で」

Claude: 「完了しました」

所要時間: 5秒
```

### 内部で何が起きているか

自然言語の指示がどのようにUnity操作に変換されるかを解説します：

```
┌─────────────────────────────────────────────────────────┐
│                   MCP経由の操作                         │
│                                                         │
│  1. Claude Code が指示を解析                           │
│     └── 「Player作成」「Rigidbody」「Collider」を識別   │
│                                                         │
│  2. 適切なAPIを選択                                     │
│     ├── unity.gameObject.create                        │
│     ├── unity.component.add (Rigidbody)                │
│     ├── unity.component.setProperty (mass=2)           │
│     ├── unity.component.add (CapsuleCollider)          │
│     └── unity.component.setProperty (height=1.8)       │
│                                                         │
│  3. MCP Bridge 経由で Unity に送信                      │
│                                                         │
│  4. Unity MCP Server が実行                             │
│                                                         │
│  5. 結果を返送                                          │
│     └── 「Playerを作成しました」                       │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 350以上のAPIでできること

UniMCP4CCは、**350以上のAPI**を提供しています。
ほぼすべてのUnity Editor操作をカバーしています。

### カテゴリ一覧

| カテゴリ | できること |
|:---|:---|
| **Scene** | 作成、保存、切り替え |
| **GameObject** | 作成、削除、検索、複製 |
| **Transform** | 位置、回転、スケール |
| **Component** | 追加、削除、プロパティ変更 |
| **Prefab** | 作成、インスタンス化、編集 |
| **Asset** | 作成、検索、インポート設定 |
| **Material** | 作成、プロパティ、テクスチャ |
| **UI** | Canvas、Button、Text |
| **Timeline** | 作成、トラック、クリップ |
| **Cinemachine** | カメラ、ブレンド、ドリー |
| **Animator** | Controller、State、Transition |
| **Physics** | Rigidbody、Collider、Joint |
| **Light** | 作成、プロパティ、Probe |
| **Editor** | Play/Stop、選択、Undo |
| **Build** | 設定、プラットフォーム |

### 具体的な使用例

自然言語での指示と、実際に呼び出されるAPIの対応を示します：

```
「Enemyというタグがついたオブジェクトを全部削除して」
→ unity.gameObject.findByTag + unity.gameObject.destroy

「画面中央にStartButtonを作成して、フォントサイズを32にして」
→ unity.ui.createButton + unity.component.setProperty

「プレイヤーを追従するカメラを作成して」
→ unity.cinemachine.createVirtualCamera + unity.cinemachine.setFollow

「夕焼けっぽいDirectional Lightを作成して」
→ unity.light.create + unity.light.setColor + unity.light.setIntensity
```

## 開発フローの変化

### 従来のフロー

従来の開発フローでは、「セットアップ」がボトルネックでした：

```
アイデア
   ↓
設計（紙に書く）
   ↓
実装（コードを書く）
   ↓
セットアップ（GUIで配置）  ← ボトルネック
   ↓
テスト
   ↓
修正 → セットアップ → テスト... ← 繰り返し
```

### MCPを使ったフロー

MCPを導入すると、実装とセットアップが同時進行になります：

```
アイデア
   ↓
Claude Code に指示
   ↓
実装 + セットアップ（同時進行）
   ↓
テスト
   ↓
「ここを修正して」→ 即反映 → テスト
```

### 変化のポイント

| 従来 | MCP後 |
|:---|:---|
| 設計→実装→配置が分離 | 一気通貫で進行 |
| 配置作業がボトルネック | 指示だけで完了 |
| 修正のたびにGUI操作 | 会話で修正 |
| 試行回数に限界 | 気軽に試せる |

## リアルタイム対話の価値

### 従来の「修正」フロー

パラメータを1つ変更するだけでも、以下の手順が必要でした：

```
1. 「ちょっと速すぎるな」と感じる
2. Unity Editorを開く
3. Hierarchyでオブジェクトを探す
4. Inspectorでスクリプトを探す
5. パラメータを見つける
6. 値を変更する
7. 再生して確認する

所要時間: 2〜3分
```

### MCPでの「修正」

MCPを使うと、対話的に修正できます：

```
あなた: 「移動速度を5から3に下げて」
Claude: 「変更しました」

所要時間: 数秒
```

### 何が変わるか

「試行のコスト」が下がると、「試行の回数」が増えます。
試行回数が増えると、品質が上がります。

```
従来:
├── 修正に2〜3分かかる
├── 面倒なので後回しにする
└── 結果として妥協が増える

MCP後:
├── 修正は数秒
├── 気になったらすぐ試す
└── 結果として品質が上がる
```

## まとめ

### MCPとは

| 項目 | 説明 |
|:---|:---|
| **定義** | AIと外部ツールを接続するオープンプロトコル |
| **策定者** | Anthropic社 |
| **例え** | USBのような標準規格 |

### Unityへの効果

| Before | After |
|:---|:---|
| GUIで手動操作 | 自然言語で指示 |
| 配置に時間がかかる | 数秒で完了 |
| 修正のたびにGUI | 会話で修正 |
| 試行回数に限界 | 気軽に何度でも |

### 提供されるAPI

- **350以上のAPI**
- Scene、GameObject、Component、Prefab、UI、Timeline...
- ほぼすべてのEditor操作をカバー

MCPは「楽をするため」の道具ではありません。
**思い描いたゲームを、諦める前に形にするためのインフラ**です。
