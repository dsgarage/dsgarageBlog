# Skills活用術：繰り返し作業を一言で

開発中に「同じような操作を何度も繰り返す」場面はありませんか？

- 「毎回同じセットアップをしている」
- 「よく使う構成を何度も説明している」
- 「定型的な作業に時間を取られている」

これを解決するのが、Claude Codeの**Skills（スキル）**機能です。

---

## Skillsとは

### 概要

**Skills**は、**よく使うワークフローを登録しておく機能**です。

一度登録すると、スラッシュコマンドで呼び出せます。

```
/setup-player
```

これだけで、複数の操作が自動実行されます。

### なぜ必要か

```
Skillsがない場合:
├── 「Playerを作成して」
├── 「CharacterControllerを追加して」
├── 「Rigidbodyを追加して」
├── 「タグをPlayerに設定して」
├── 「レイヤーをCharacterに設定して」
└── 毎回5回の指示が必要

Skillsがある場合:
└── /setup-player で完了
```

## Unity向けSkill例

### 1. /setup-player

**目的**: プレイヤーオブジェクトの基本セットアップ

```markdown
# /setup-player

Playerオブジェクトを一式作成します。

## 手順

1. 空のGameObject「Player」を作成
2. CharacterControllerを追加
   - height: 1.8
   - radius: 0.3
   - center: (0, 0.9, 0)
3. Rigidbodyを追加
   - isKinematic: true
   - useGravity: false
4. タグを「Player」に設定
5. レイヤーを「Character」に設定
6. 子オブジェクト「Model」を作成（モデル配置用）
7. 子オブジェクト「CameraTarget」を作成
   - position: (0, 1.5, 0)

## 完了後

Playerの基本構成が完了。PlayerControllerスクリプトの追加を検討してください。
```

### 2. /create-ui

**目的**: UI階層を一括構築

```markdown
# /create-ui

基本的なUI階層を構築します。

## 手順

1. Canvas「MainCanvas」を作成
   - Render Mode: Screen Space - Overlay
   - Canvas Scaler:
     - UI Scale Mode: Scale With Screen Size
     - Reference Resolution: 1920 x 1080
     - Match: 0.5

2. 子オブジェクト構成:
   MainCanvas
   ├── Background     # 背景用
   ├── GameUI         # ゲーム中UI
   ├── MenuUI         # メニューUI
   ├── PopupUI        # ポップアップ用
   └── SystemUI       # システムメッセージ用

3. EventSystemがなければ作成

## 使い方

各カテゴリ内にUI要素を配置してください。
```

### 3. /add-physics

**目的**: 物理オブジェクトの一括設定

```markdown
# /add-physics [target]

指定したオブジェクトに物理コンポーネントを追加します。

## パラメータ

- target: 対象オブジェクト名（デフォルト: 選択中のオブジェクト）

## 手順

1. Rigidbodyを追加
   - mass: 1
   - drag: 0.5
   - angularDrag: 0.05

2. MeshColliderまたはBoxColliderを追加
   - MeshFilterがあれば MeshCollider（Convex）
   - なければ BoxCollider

3. PhysicMaterialを設定
   - dynamicFriction: 0.6
   - staticFriction: 0.6
   - bounciness: 0.2

## 完了後

物理シミュレーション対応完了。massや摩擦は必要に応じて調整してください。
```

### 4. /create-enemy

**目的**: 敵オブジェクトのテンプレート作成

```markdown
# /create-enemy [name]

敵オブジェクトのテンプレートを作成します。

## パラメータ

- name: 敵の名前（デフォルト: Enemy）

## 手順

1. 空のGameObject「{name}」を作成
2. 以下のコンポーネント構成:
   - CapsuleCollider (trigger: true)
   - Rigidbody (isKinematic: true)
   - NavMeshAgent (オプション)

3. タグを「Enemy」に設定
4. レイヤーを「Enemy」に設定

5. 子オブジェクト構成:
   {name}
   ├── Model        # 見た目
   ├── HitBox       # 攻撃判定
   └── DetectArea   # 検知範囲

## 完了後

EnemyControllerスクリプトを追加してください。
```

### 5. /setup-camera

**目的**: カメラシステムの構築

```markdown
# /setup-camera

Cinemachineを使用したカメラシステムを構築します。

## 前提

- Cinemachineパッケージがインストール済み

## 手順

1. CinemachineBrain付きのMain Cameraを確認/作成

2. 以下のVirtual Cameraを作成:

   CM_Follow (追従カメラ)
   - Body: Framing Transposer
   - Aim: Composer
   - Priority: 10

   CM_FreeLook (フリールック)
   - 3リグ設定
   - Priority: 5

   CM_Cutscene (カットシーン用)
   - Priority: 0

3. カメラグループ「Cameras」を作成し、配下に配置

## 使い方

「CM_FollowのFollowをPlayerに設定して」
「CM_FreeLookのLookAtをPlayer/CameraTargetに設定して」
```

### 6. /cleanup-scene

**目的**: シーンの整理

```markdown
# /cleanup-scene

現在のシーンを整理します。

## 手順

1. Hierarchyのセクション区切りを確認
   ---Environment---
   ---Characters---
   ---UI---
   ---System---
   なければ作成

2. 未分類オブジェクトを検出

3. 空のGameObjectを検出（Transform以外のコンポーネントなし）

4. 無効なコンポーネント（Missing）を検出

5. レポート作成

## 出力例

[整理レポート]
- 未分類オブジェクト: 3件
  - Cube
  - Sphere
  - Temp

- 空のGameObject: 2件
  - GameObject (1)
  - NewObject

- Missingコンポーネント: 1件
  - Player (MissingScript)
```

## Skillの登録方法

### 方法1: .claude/skills/ フォルダ

プロジェクトルートに `.claude/skills/` フォルダを作成し、Markdownファイルを配置します。

```
プロジェクト/
├── .claude/
│   └── skills/
│       ├── setup-player.md
│       ├── create-ui.md
│       └── add-physics.md
├── Assets/
└── ...
```

### ファイル形式

```markdown
# /skill-name

スキルの説明

## 手順

1. ステップ1
2. ステップ2
...
```

### 方法2: グローバルSkills

ユーザー共通のSkillsは `~/.claude/skills/` に配置します。

```
~/.claude/
└── skills/
    ├── setup-unity-project.md
    └── check-naming.md
```

## カスタムSkillの作り方

### ステップ1: 繰り返しパターンを見つける

```
過去1週間で何度も行った操作:
├── UIボタンの作成（5回）
├── 敵オブジェクトの作成（8回）
├── シーン整理（3回）
└── デバッグ用オブジェクト作成（10回）
```

### ステップ2: 手順を書き出す

```markdown
# /create-debug-object

デバッグ用の可視化オブジェクトを作成

## 手順

1. Cubeを作成
2. 名前を「DEBUG_{timestamp}」に設定
3. スケールを(0.1, 0.1, 0.1)に設定
4. マテリアルを赤の半透明に設定
5. タグを「EditorOnly」に設定（ビルドから除外）
```

### ステップ3: パラメータを追加

```markdown
# /create-debug-object [position] [color]

デバッグ用の可視化オブジェクトを作成

## パラメータ

- position: 配置位置（デフォルト: 0, 0, 0）
- color: 色（デフォルト: 赤）

## 手順
...
```

### ステップ4: テスト

```
/create-debug-object (0, 1, 0) 青
```

期待通りに動作するか確認。

## チームでの共有

### Git管理

`.claude/skills/` をGit管理することで、チーム全体でSkillsを共有できます。

```gitignore
# .gitignore
# skillsは除外しない（共有する）
!.claude/skills/
```

### 命名規則

チームで使う場合は、命名規則を決めておく:

```
/unity-*       # Unity固有の操作
/debug-*       # デバッグ用
/setup-*       # セットアップ系
/create-*      # 作成系
/check-*       # 確認・検証系
```

### ドキュメント化

```markdown
# チームSkills一覧

| コマンド | 説明 | 作者 |
|:---|:---|:---|
| /setup-player | プレイヤー作成 | @tanaka |
| /create-ui | UI階層構築 | @suzuki |
| /add-physics | 物理設定 | @yamada |
```

## Skillsのベストプラクティス

### 1. 小さく保つ

```markdown
# 悪い例
/setup-entire-game  # 1つで全部やろうとする

# 良い例
/setup-player
/setup-camera
/create-ui
/setup-lighting
```

### 2. パラメータを活用

```markdown
# 悪い例
/create-enemy-slime
/create-enemy-goblin
/create-enemy-dragon

# 良い例
/create-enemy slime
/create-enemy goblin
/create-enemy dragon
```

### 3. 説明を丁寧に

```markdown
# 悪い例
プレイヤーを作る

# 良い例
## 目的
プレイヤーオブジェクトの基本構成を作成します。

## 作成されるもの
- Player (GameObject)
  - CharacterController
  - Rigidbody (Kinematic)
- 子オブジェクト: Model, CameraTarget

## 完了後の作業
PlayerControllerスクリプトを追加してください。
```

### 4. エラーハンドリング

```markdown
## 前提条件

- Playerタグが定義済みであること
- Characterレイヤーが定義済みであること

## エラー時

タグまたはレイヤーが未定義の場合:
1. Project Settings > Tags and Layers を開く
2. 必要なタグ/レイヤーを追加
3. 再度Skillを実行
```
