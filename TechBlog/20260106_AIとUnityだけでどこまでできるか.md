# AIとUnityだけでどこまでできるか

「AIでUnity開発」と聞くと、どこまでできるのか気になりませんか？

本記事では、**MCP（Model Context Protocol）を使ってAIとUnityだけで何ができるのか**を、具体的なAPI一覧と実例で紹介します。

---

## CC2UniMCPの規模

### 数字で見る

| 項目 | 数値 |
|:---|:---|
| **総API数** | 350以上 |
| **カテゴリ数** | 15+ |
| **対応Unityバージョン** | 2021.3 LTS〜 |

### 対応範囲

```
Unity Editorで日常的に行う操作の約80%をカバー
```

つまり、**ほとんどの作業を自然言語で指示可能**です。

## できないこと・現在の限界

すべてができるわけではありません。

| できないこと | 理由 |
|:---|:---|
| **Shader Graphのノード編集** | ビジュアルエディタの内部操作 |
| **アニメーションカーブの細かい編集** | キーフレーム単位は限定的 |
| **Visual Scripting** | ノードベースの操作 |
| **カスタムEditorウィンドウ操作** | 標準外のGUI |
| **サードパーティツールの操作** | 対応API不足 |

### ただし...

これらも**C#スクリプトを生成してもらう**ことで間接的に対応可能です。

```
「Shader Graphで使うカスタムノードのスクリプトを生成して」
→ スクリプトを生成してもらい、自分でインポート
```

## 実際の開発フロー比較

### ケース1: キャラクターのセットアップ

**従来のフロー:**
```
1. Hierarchyで右クリック → Create Empty (5秒)
2. 名前を「Player」に変更 (3秒)
3. Add Component → Character Controller (10秒)
4. Inspectorでパラメータ設定 (20秒)
5. Add Component → Rigidbody (5秒)
6. Kinematicにチェック (3秒)
7. 子オブジェクト作成 × 3 (30秒)
8. 各種設定... (60秒)

合計: 約2〜3分
```

**MCPフロー:**
```
「Playerオブジェクトを作成して。
  CharacterController（height: 1.8）と
  Rigidbody（Kinematic）を追加。
  子オブジェクトにModel、CameraTarget、Weaponを作成」

合計: 約10秒
```

### ケース2: UIの構築

**従来のフロー:**
```
1. Canvas作成
2. Canvas Scalerの設定
3. 背景用Panel
4. ボタン作成
5. テキスト調整
6. アンカー設定...

合計: 5〜10分
```

**MCPフロー:**
```
「メインメニュー用のCanvasを作成して。
  1920x1080基準でScale With Screen Size。
  Start、Options、Exitの3つのボタンを縦に並べて」

合計: 約15秒
```

### ケース3: レベルデザインの調整

**従来のフロー:**
```
1. オブジェクトを選択
2. 位置を微調整
3. 別のオブジェクトを選択
4. 位置を微調整...
（これを何十回も繰り返し）

合計: 数十分〜数時間
```

**MCPフロー:**
```
「Platformタグのオブジェクトを全部選んで、
  Y座標を一律0.5メートル上げて」

「Obstacleを等間隔（5メートル間隔）に再配置して」

合計: 数秒〜数分
```

## まとめ

### AIとUnityで変わること

```
Before:
├── コードを書く → 手動で配置 → テスト
├── 「面倒だからあとで」が増える
└── アイデアを諦めることが多い

After:
├── 指示するだけ → 自動で配置 → すぐテスト
├── 「試してみよう」が増える
└── アイデアを形にできる
```

## おわりに

「AIにUnityでゲームを作ってもらう」

この夢のような話が、2025年現在、**かなりの部分で実現可能**になっています。

MCPは「楽をするため」の道具ではありません。
**思い描いたゲームを、諦める前に形にするためのインフラ**です。

試すコストが下がれば、試す回数が増える。
試す回数が増えれば、良いゲームができる。

ぜひ、この新しい開発スタイルを試してみてください。
