# Claude Codeとは何か

「AIにコードを書いてもらう」——ChatGPTやCopilotでそれが可能になりました。
しかし、実際に使ってみると気づくことがあります。

**コードは生成できるが、プロジェクト全体を理解してくれない。**

ChatGPTにコードを貼り付け、修正してもらい、結果をコピペで戻す。
この往復作業が、開発効率を大きく下げています。

本記事では、この問題を解決する**Claude Code**について解説します。
「なぜClaude Codeが他のAIツールと根本的に異なるのか」を理解することで、適切なツール選択ができるようになります。

:::message
**Unity開発者の方へ**

Claude Codeは強力なツールですが、Unity Editorを直接操作することはできません。
この問題を解決するのが**UniMCP4CC**（Unity MCP Server for Claude Code）です。

- GitHub: [dsgarage/UniMCP4CC](https://github.com/dsgarage/UniMCP4CC)
- 対応Unity: 2021.3 LTS以降
- ライセンス: MIT
:::

## Claude Codeとは

### 従来のAIアシスタントの限界

従来のAIアシスタント（ChatGPT、Gemini等）でコードを書く場合、以下の問題があります：

```
1. ChatGPTにコードを貼り付ける
2. 「これを修正して」と依頼する
3. 修正されたコードが表示される
4. コードをコピーする
5. エディタに戻って貼り付ける
6. 動かない → 1に戻る

※ 往復の手間がかかる
※ AIはプロジェクト全体を知らない
```

この往復作業が、開発効率を大きく下げています。

### Claude Codeの解決策

**Claude Code**は、Anthropic社が開発した**開発エージェント**です。
単なるチャットボットではありません。

ターミナルから起動し、プロジェクトディレクトリに「住み着いて」開発を支援します：

```
$ cd my-project
$ claude

# Claudeはこの時点で以下を把握します：
# - フォルダ構造
# - 各ファイルの内容
# - 使用している言語・フレームワーク
# - 依存関係
```

## 従来のAIとの根本的な違い

### アーキテクチャの違い

```
┌─────────────────────────────────────────────────────┐
│              従来のAIアシスタント                    │
│                                                     │
│  ┌─────────┐         ┌─────────────────┐          │
│  │  人間   │ ──────→ │  AI (チャット)  │          │
│  └─────────┘  質問    └─────────────────┘          │
│       ↑                      │                     │
│       └──────────────────────┘                     │
│              回答（テキスト）                       │
│                                                     │
│  ※ AIはプロジェクトの中身を知らない               │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│                 Claude Code                         │
│                                                     │
│  ┌─────────┐         ┌─────────────────┐          │
│  │  人間   │ ──────→ │  Claude Code    │          │
│  └─────────┘  指示    │                 │          │
│       ↑               │  ┌───────────┐ │          │
│       │               │  │プロジェクト│ │          │
│       │               │  │ファイル群 │ │          │
│       │               │  └───────────┘ │          │
│       └───────────────┴─────────────────┘          │
│         実際にファイルを読み書き・実行              │
│                                                     │
│  ※ AIがプロジェクト全体を把握している             │
└─────────────────────────────────────────────────────┘
```

### 作業フローの違い

**ChatGPTでコードを書く場合：**

往復作業が必要で、AIはその都度コンテキストを失います。

**Claude Codeでコードを書く場合：**

```
1. ターミナルで claude を起動
2. 「これを修正して」と依頼
3. ファイルが直接更新される
4. 動かない → 「エラーが出た」と伝える → 自動修正

※ 往復なし、その場で完結
```

## Claude Codeの主要機能

### 1. プロジェクト全体を読み込む

Claude Codeは、起動したディレクトリ内の**すべてのファイルを読み込み、内容を把握**します。
これにより、プロジェクト全体の文脈を理解した上でコードを生成できます。

### 2. 複数ファイルを同時に変更

単一ファイルの編集ではなく、**プロジェクト全体を見渡した変更**が可能です：

```
あなた: 「ユーザー認証機能を追加して」

Claude Code:
├── src/auth/AuthService.ts    [新規作成]
├── src/auth/AuthContext.tsx   [新規作成]
├── src/components/Login.tsx   [新規作成]
├── src/App.tsx                [修正]
└── package.json               [依存追加]
```

### 3. ターミナル操作の実行

Git操作やビルドコマンドも実行できます：

```
あなた: 「変更をコミットして」

Claude Code:
$ git add .
$ git commit -m "Add user authentication feature"
```

### 4. 大規模コンテキスト

100kトークン以上のコードを一度に処理できます。
大規模プロジェクトでも、文脈を見失いません。

## 他のAIツールとの比較

### 比較表

| 項目 | ChatGPT | Claude Code |
|:---|:---|:---|
| **提供形態** | Webブラウザ / API | CLI（ターミナル） |
| **ファイル操作** | 手動コピペ | 自動で読み書き |
| **コンテキスト長** | 32k〜128kトークン | 100k〜200kトークン級 |
| **プロジェクト理解** | 貼り付けた部分のみ | 全体を自動把握 |
| **コマンド実行** | 不可 | git, npm, build等 |
| **複数ファイル変更** | 1ファイルずつ | 一括で複数ファイル |
| **料金** | 月額$20〜 | 従量課金（API） |
| **得意分野** | 解説、学習、単発質問 | 実際の開発作業 |

### 使い分けの指針

それぞれのツールには適した場面があります：

```
ChatGPTを使うべき場面:
├── 概念を理解したい（「Reactのライフサイクルとは？」）
├── アルゴリズムの説明が欲しい
├── コードレビューの意見が欲しい
├── 複数の選択肢を比較したい
└── ドキュメントを読んでもらいたい

Claude Codeを使うべき場面:
├── 実際にコードを書きたい
├── 複数ファイルにまたがる変更
├── リファクタリング
├── テストの追加
└── バグの調査と修正
```

### GitHub Copilotとの違い

GitHub Copilotは「リアルタイム補完」に特化したツールです：

| 項目 | Copilot | Claude Code |
|:---|:---|:---|
| 動作方式 | リアルタイム補完 | 対話型エージェント |
| コンテキスト | 編集中ファイル周辺 | プロジェクト全体 |
| 得意分野 | 定型コード、ボイラープレート | 大規模改変、設計 |

**比喩で説明すると：**
- Copilot = 「文章の続きをその場で執筆してくれる優秀な秘書」
- Claude Code = 「設計図を描き現場監督もしてくれる共同開発者」

## CLAUDE.mdファイル

### なぜ重要なのか

Claude Codeの最も重要な機能の一つが、**CLAUDE.md**ファイルです。

プロジェクトルートにこのファイルを配置すると、Claude Codeは**プロジェクト固有のルール**を理解します。
これにより、毎回同じ説明を繰り返す必要がなくなります。

```
CLAUDE.mdなし:
├── 毎回同じ説明をする必要がある
├── AIが勝手な判断をする
└── プロジェクトのルールを無視される

CLAUDE.mdあり:
├── ルールが自動的に適用される
├── 一貫したコードが生成される
└── レビュー負担が減る
```

### 基本構成

```markdown
# プロジェクト概要

このプロジェクトは○○です。

## Code Style（コードスタイル）

- TypeScriptを全てに用いる（例外なし）
- Reactは関数コンポーネント + フックのみ使用
- インデントはスペース2つ

## Architecture Notes（アーキテクチャ方針）

- 状態管理にはZustandを使用
- API呼び出しは /src/utils/api.ts のラッパー経由

## Don't Do（禁止事項）

- クラスコンポーネントは使わない
- any型は使用しない
- console.logを本番コードに残さない
```

## プランモード

### 設計を先に確認する

Claude Codeには**プランモード**という機能があります。
コードを実行・変更せずに、まず**設計プランを提案**させるモードです。

```
Shift + Tab を2回押す → プランモードON
```

### なぜ重要なのか

```
プランモードなし:
├── AIがいきなりコードを書き始める
├── 意図と違う実装が進む
└── 後から大幅な修正が必要

プランモードあり:
├── 設計を事前に確認できる
├── 方向修正が容易
└── 無駄な実装を防げる
```

### 使用例

```
[プランモード ON]

あなた: 「ユーザープロフィール編集機能を追加したい」

Claude Code:
「以下の設計を提案します：

1. データモデル
   - UserProfile型を定義
   - 必要なフィールド: name, email, avatar, bio

2. コンポーネント構成
   - ProfileEditForm: メインのフォーム
   - AvatarUploader: 画像アップロード
   - ProfilePreview: プレビュー表示

この設計で進めてよろしいですか？」

あなた: 「avatarは後で対応するので、まずは基本情報だけで」

Claude Code: 「承知しました。avatar関連を除いた設計で進めます」
```

## 実際の使用例

### 例1: リファクタリング

大規模なコード分割も自動で行えます：

```
あなた: 「src/utils/helpers.tsが500行を超えている。
        責務ごとにファイルを分割して」

Claude Code:
「以下のように分割します：

├── src/utils/
│   ├── helpers/
│   │   ├── string.ts    (文字列操作)
│   │   ├── date.ts      (日付操作)
│   │   ├── array.ts     (配列操作)
│   │   ├── validation.ts (バリデーション)
│   │   └── index.ts     (再エクスポート)

既存のimport文も自動で更新します。
実行してよろしいですか？」
```

### 例2: バグ調査

問題の原因調査も任せられます：

```
あなた: 「本番環境でユーザーがログアウトされる問題が起きている。
        原因を調査して」

Claude Code:
「コードを確認しました。

原因の可能性：
1. AuthContext.tsx:45 - トークン更新時にstateが初期化されている
2. api.ts:23 - 401エラー時に無条件でlogout()を呼んでいる
3. localStorage.ts:12 - JSON.parseでエラー時に空オブジェクトを返している

最も可能性が高いのは2番です。
トークンの有効期限切れと認証エラーを区別していません。

修正案を提示しますか？」
```

## プロンプトの書き方

### 悪いプロンプト

```
「ログイン機能を作って」
```

これだけでは、AIは以下を判断できません：
- どの認証方式を使うか
- セッション管理はどうするか
- UIはどのようなものか

### 良いプロンプト

```
「ログイン機能を作って。

要件:
- メール/パスワード認証
- JWTトークンをlocalStorageに保存
- 認証失敗時はフォーム上にエラー表示
- 成功時は/dashboardにリダイレクト

制約:
- 既存のAuthContextを使用
- Tailwind CSSでスタイリング
- バリデーションはzodを使用」
```

具体的な要件と制約を明示することで、意図通りのコードが生成されます。

## まとめ

### Claude Codeとは

| 特徴 | 説明 |
|:---|:---|
| **開発エージェント** | 単なるチャットではなく、実際に開発作業を行う |
| **プロジェクト全体理解** | フォルダ構造、依存関係を自動把握 |
| **ファイル直接操作** | コピペ不要、直接読み書き |
| **コマンド実行** | git、npm、build等を実行 |

### 重要な機能

- **CLAUDE.md**: プロジェクトルールを共有し、一貫したコード生成
- **プランモード**: 実装前に設計を確認し、手戻りを防止

### 他ツールとの使い分け

- **ChatGPT**: 概念理解、単発質問に最適
- **Copilot**: 日常的なコード補完に最適
- **Claude Code**: プロジェクト全体の開発に最適
